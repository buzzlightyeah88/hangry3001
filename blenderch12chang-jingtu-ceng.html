BlenderCH12場景、圖層

當你處理大型或複雜的 3D 場景時，合理的場景組織和圖層管理對於工作流程至關重要。Blender 提供了強大的集合（Collections）、命名與篩選功能，以及圖層與場景管理工具，可以讓你高效管理和控制大型場景。下面是更詳細的步驟，帶有所有關鍵位置和快捷鍵的英文描述，適用於多種工作環境。

1. 集合（Collections）：如何組織場景中的物件

1.1 什麼是集合（Collections）？

在 Blender 中，集合是一種用來組織物件的容器，可以包含不同類型的物件（例如模型、燈光、相機等）。它使得管理大型場景變得更簡單，能夠讓你隨時顯示、隱藏、選擇或編輯一組物件。

1.2 創建與管理集合

  • 創建新的集合：

    1. 在 Outliner（場景大綱）面板中，右鍵點擊一個物件，選擇 Move to Collection（移動到集合）。
    2. 在彈出的菜單中，選擇 New Collection（新建集合），並命名為適合的名稱，例如 Furniture（家具）或 Lighting（燈光）。
    3. 或者，點擊頂部菜單的 Collection → New 來創建集合（Shift + C 是快速創建集合的快捷鍵）。

  • 將物件移入集合：

    1. 在 3D View（3D 視圖）中選擇你要移動的物件。
    2. 按 M，調出 Move to Collection 菜單，然後選擇你希望將物件移入的集合。
    3. 你可以選擇將物件放入已經存在的集合中，或者創建一個新的集合。

  • 集合的層級結構管理：

    • 你可以在 Outliner 中創建嵌套集合。例如，將一個集合命名為 Environment，並將其他子集合如 Trees、Ground 放在其中。這樣有助於讓場景結構更有條理。
    • 你可以通過拖放操作將子集合嵌套到主集合中。

1.3 使用集合進行顯示、隱藏和渲染控制

  • 顯示與隱藏集合：

    1. 在 Outliner 中，每個集合旁邊會有一個眼睛圖標。點擊這個眼睛圖標，可以隱藏或顯示該集合中的物件。
    2. 這可以幫助你在處理場景時，專注於特定部分。例如，隱藏家具集合來專注於建築結構。

  • 禁用集合渲染：

    1. 在 Outliner 中，集合旁邊有一個相機圖標。點擊它會禁用該集合的渲染。
    2. 這樣可以在渲染最終圖像時不顯示某些集合中的物件，常用於不需要渲染的場景元素（例如輔助物件或參考物件）。

2. 命名與篩選：命名物件及使用過濾器管理大場景

2.1 命名物件

在大型場景中，正確命名物件對於快速識別和選擇物件至關重要。這不僅有助於你在場景中尋找物件，還有助於在大型專案中保持一致性。

  • 命名規範：

    1. 給物件賦予描述性的名稱，例如將 Cube 改名為 Chair_01、Tree_01 等。
    2. 使用數字、字母或類別來區分相同類型的物件，例如 Car_01、Car_02，或者 Door_Wood 和 Door_Metal。

  • 批量重命名物件：

    1. 在 Outliner 中右鍵點擊任意物件，選擇 Rename（重命名），然後手動輸入新名稱。
    2. 你可以選擇性地使用批量命名工具。使用 Ctrl + Shift + R 可以在一個範圍內重命名物件。

2.2 使用篩選器篩選物件

Blender 提供了多種篩選工具，能幫助你在擁有大量物件的場景中快速定位所需的物件。

  • 篩選器功能：

    1. 在 Outliner 上方有一個篩選器欄位。在這裡，你可以輸入物件名稱來篩選顯示物件。比如輸入 Cube 會顯示所有名稱中包含 Cube 的物件。
    2. 在篩選器欄中點擊 Filter（過濾器）圖標，選擇顯示/隱藏物件、集合、燈光等。

  • 使用類型過濾器：

    1. 在 Outliner 的過濾選項中，點擊下拉菜單，選擇物件類型，例如 Mesh、Camera、Light，只顯示這些類型的物件。

  • 快速過濾物件：

    1. 使用 Shift + G 可以根據不同屬性快速選擇物件，例如選擇所有使用相同材質的物件，或者所有處於同一集合中的物件。
    2. 在 3D View 中，選擇一個物件後，按 Ctrl + L 來選擇與之相連的物件，這樣能在場景中快速選擇並修改相關物件。

3. 圖層和場景功能：多場景和圖層的使用方法

3.1 圖層管理（Layers）

在 Blender 2.8 及以後的版本中，圖層管理已經被集合（Collections）取代。但有時候，你仍然需要知道如何處理舊版圖層（例如當你加載舊場景時）。如果你使用的是 Blender 2.8 或更高版本，圖層將以集合的形式出現。

  • 在集合中使用圖層：
    1. 你可以在 Outliner 中組織物件，使它們屬於不同的集合，從而模擬多圖層的效果。
    2. 每個集合可以看作是一個圖層，它可以包含多個物件，並且可以單獨控制顯示、隱藏、選擇和渲染。

3.2 多場景（Multiple Scenes）

Blender 允許在同一個文件中創建多個場景。這對於處理大型專案（如動畫、不同渲染版本等）時非常有用。

  • 創建新場景：

    1. 點擊 File（文件） → New → Scene（新建場景），可以創建一個新的場景。
    2. 你也可以選擇基於當前場景創建一個新場景，這樣新場景會繼承當前的設置（例如物件、材質等）。
    3. 使用 Ctrl + Shift + N 來創建新的場景。

  • 切換場景：

    1. 在 Outliner 中，你會看到場景的列表。點擊任何一個場景名稱來切換到該場景。
    2. 你可以在不同場景之間來回切換，並且它們的設定和物件是獨立的。

3.3 場景間物件共享與管理

  • 物件共享：

    1. 在多場景工作中，你可以使用 Link 和 Append 操作來共享物件。
    2. Link 允許你將其他場景中的物件鏈接到當前場景，這樣當物件更新時，所有場景都會反映更改。
    3. Append 則是將物件實際複製到當前場景中，使其變成獨立的副本。

  • 場景參數設置：每個場景都有自己的屬性，你可以在 Properties Panel（屬性面板）中設置不同的渲染選項、相機、燈光等。使用 Scene 標籤來修改場景特定的屬性。

4. 進階功能與技巧

4.1 使用視圖層（View Layers）來分層管理

視圖層（View Layers）是 Blender 中處理複雜渲染的強大工具。它允許你將場景中的物件按照不同的視角進行分層管理，每個視圖層可以有不同的顯示、渲染和遮罩設置。

  • 創建視圖層：

    1. 在 Properties 面板中，找到 View Layer 標籤（它是類似於「攝影機」的圖標）。
    2. 在 View Layer 設置中，點擊 + 創建新的視圖層。
    3. 每個視圖層都可以選擇不同的物件、集合、燈光，甚至是渲染設置來創建不同的渲染效果。

  • 視圖層與集合的結合使用：

    1. 你可以將特定的集合分配到不同的視圖層，這樣每個視圖層僅渲染特定集合中的物件。
    2. 這對於渲染特效、遮罩層、通道等非常有用，能夠讓你在合成過程中有更多的控制權。

  • 遮罩（Masks）：在視圖層中，你可以使用遮罩來控制物件的顯示。例如，你可以將某些物件或集合的顯示狀態設置為遮罩，這樣它們在渲染中不會顯示出來。

4.2 利用標籤（Tags）來標記物件

Blender 允許使用標籤（Tags）來對物件進行額外分類，這有助於更精細的管理。標籤通常在大型場景中很有用，可以快速標記物件屬性或用途。

  • 為物件添加標籤：

    1. 在 Outliner 中右鍵點擊物件，選擇 Add Tag（添加標籤）。
    2. 輸入標籤名稱，例如 Prop（道具）、Character（角色）等。
    3. 你可以為每個物件添加多個標籤來標記其屬性。

  • 篩選帶標籤的物件：

    1. 在 Outliner 中，點擊篩選器（Filter）圖標，並選擇 Tags，然後選擇你想要篩選的標籤。
    2. 這樣，所有擁有該標籤的物件會被高亮顯示，幫助你快速查找。

4.3 自訂快捷鍵與快捷操作

Blender 提供了大量的快捷鍵來加快工作流程。你可以自定義快捷鍵來提高場景組織與圖層管理的效率。

  • 自定義快捷鍵：

    1. 前往 Edit → Preferences → Keymap，在這裡你可以查看、修改並創建新的快捷鍵。
    2. 例如，為經常使用的命令（如 Move to Collection）設置一個自定義的快捷鍵，這樣你不需要每次都進入菜單。

  • 場景/集合選擇快捷鍵：

    1. 當場景中有多個集合時，使用 Ctrl + [數字鍵] 可以快速切換顯示不同的集合。
    2. 使用 Shift + [數字鍵] 可以選擇不同的視圖層。

4.4 使用 Python 腳本進行自動化

Blender 提供了強大的 Python API，允許你使用腳本進行自動化操作。這對於管理大型專案中的集合、物件和場景非常有用。

  • 使用腳本來自動化集合管理：
    1. 打開 Scripting（腳本）視窗，創建新的 Python 腳本。
    2. 使用 bpy（Blender Python 模組）來創建、管理和組織集合。例如，下面的腳本可以自動創建集合並將物件移入其中：
python
import bpy

# 創建新集合
collection = bpy.data.collections.new("MyCollection")
bpy.context.scene.collection.children.link(collection)

# 移動物件到新集合
obj = bpy.context.view_layer.objects.active
collection.objects.link(obj)
  • 批量重命名物件： 你可以編寫腳本來根據規則批量重命名物件，這對於大型場景中的物件管理非常有用。
python
 
import bpy

# 批量重命名物件
for i, obj in enumerate(bpy.context.selected_objects):
    obj.name = f"Object_{i+1}"

4.5 使用軟體連接和協作

Blender 支援多種協作方式，尤其是當你與團隊合作時。以下是一些協作工具：

 

      • 使用外部資源：

        1. 使用 Link（鏈接）而非 Append（追加）將外部資源引入場景，這樣可以保持物件的更新同步。
        2. 使用 Asset Browser（資產瀏覽器）來管理資源，將常用的物件、材質、燈光等資源保存為資產，方便在不同場景中重複使用。

      • 利用版本控制：使用 Git 或其他版本控制系統來管理 Blender 專案。這樣，即使場景文件變得龐大，也能保持對不同版本的控制，並確保團隊成員之間的協作順利進行。
